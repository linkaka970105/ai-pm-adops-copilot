为移动互联网广告行业的广告运营、客户成功、广告优化师和技术支持人员提供的一款 AI 智能排障助手（内嵌于投放系统后台），通过 RAG 知识库问答、历史工单检索、广告指标拆解和数据分析 Agent，帮助他们快速定位投放异常、归因差异、素材审核失败和 SDK 接入问题，从而实现问题处理效率提升、客户响应速度提升和团队经验沉淀。

### 目标用户画像

#### 广告运营

- **基本特征：**22\-35 岁，通常有 0\.5\-5 年广告投放或运营经验

- **工作/生活场景：**搭建广告计划、配置 Campaign、检查素材、排查消耗异常、处理客户问题、对接技术和平台。每天查看大量账户、Campaign、素材、国家、广告位维度的数据

- **核心痛点与诉求：**问题多、系统多、文档多、客户催得急，需要快速判断问题原因，知道下一步该查什么、改什么、问谁

#### 客户成功

- **基本特征：**22\-38 岁，负责广告主或流量主客户服务

- **工作/生活场景：**回复客户问题、解释数据差异、同步投放效果、协调内部资源。

- **核心痛点与诉求：**对技术细节不够了解，但必须快速给客户专业回复，需要快速生成准确、专业、可解释的客户回复内容

#### 广告优化师

- **基本特征：**22\-38 岁，负责广告主侧买量和预算增长

- **工作/生活场景：**控制 CPI、CPA、ROAS、留存、LTV，判断预算是否加减。每天看广告后台、MMP、BI、素材报表，判断哪里出了问题

- **核心痛点与诉求：**数据很多，定位问题慢；平台报表和内部数据经常对不上，需要关注指标涨跌，知道“为什么变差”和“该怎么办”

#### 技术支持

- **基本特征：**22\-38 岁，熟悉 SDK、API、日志、归因回传

- **工作/生活场景：**排查广告展示、点击、回传、SDK 接入、事件上报、日志异常。运营或客户反复询问技术问题，需要人工看日志、查文档

- **核心痛点与诉求：**大量重复问题占用技术时间，很多问题其实 SOP 可以解决。减少重复咨询，把简单问题前置给 AI 处理，复杂问题再升级人工

## **业务背景与问题定义**

### 现有业务/行业背景

> **说明：** 描述你产品将要切入的行业或业务领域的现状。这个领域的核心业务流程是什么？关键的商业模式是什么？当前面临的主要挑战或发展趋势是什么？
> 
> 

- **行业整体现状**：移动互联网广告行业是连接广告主增长预算与媒体流量库存的效果型营销市场，覆盖 App 广告、Web 广告、信息流广告、程序化广告、OEM/On\-Device 广告等场景。行业的核心交易对象并不只是曝光或点击，而是通过数据、算法、创意和归因体系，将用户注意力转化为可衡量的安装、注册、下单、付费、留存和 ROAS 等商业结果。

- **核心业务流程**：广告主提出投放目标和预算，广告运营或投放平台创建广告计划，上传素材并设置定向、出价、转化事件和归因链接；广告平台通过 DSP、ADX、SSP、广告 SDK 或 OEM 渠道完成广告请求、实时竞价和展示；用户看到广告后产生点击、下载、注册、付费等行为；MMP、广告平台和广告主 BI 系统分别记录归因和转化数据；运营人员再根据 CPI、CPA、CTR、CVR、ROAS、LTV 等指标持续优化预算、素材、出价和流量结构。

- **商业模式**：一类是广告主侧的效果投放模式，即广告主按照 CPM、CPC、CPI、CPA、ROAS 等方式为流量和转化付费，广告平台、代理商或广告网络从中获得服务费、技术费或流量差价；另一类是流量主侧的广告变现模式，即 App、网站或 OEM 厂商通过接入广告 SDK、广告聚合平台或系统级广告位，将用户流量出售给广告主，从而获得广告收入分成。

- **行业挑战**：投放链路长、参与方多，导致问题定位复杂；广告平台、MMP 和广告主内部 BI 的数据口径不一致，归因解释成本高；隐私政策收紧后，用户级追踪能力下降，投放优化和转化归因更依赖模型和一方数据；素材审核、SDK 接入、事件回传、广告消耗异常等问题高频发生，但相关知识分散在平台文档、内部 SOP、历史工单和员工经验中，导致运营和客户成功团队处理问题时高度依赖人工排查。

- **发展趋势**：移动广告行业正在从“人工经验驱动”转向“数据与 AI 驱动”。广告平台越来越依赖自动出价、智能预算分配、AIGC 创意生成和自动化投放系统；广告主也更加关注投放效率、归因准确性、客户增长质量和团队协作效率。因此，在广告运营排障、数据口径解释、素材审核判断、投放异常诊断等高频场景中，引入 AI 智能问答和排障助手具有较强的业务价值。该产品正是切入移动互联网广告投放运营中的“问题排查与知识复用”环节，帮助团队降低沟通成本、提升响应速度，并沉淀可复用的专家经验。

### 核心问题识别

> **说明：** 基于上述背景，精准定位你要解决的核心问题。这个问题应该是具体的，而不是空泛的。
> 
> 

**问题一：知识分散，信息检索成本高**

广告行业内部知识通常分散在多个地方：

```Plain Text
飞书文档
企业微信聊天记录
Notion / Confluence
Google Doc
平台帮助中心
SDK接入文档
MMP归因文档
历史工单
BI看板
邮件
Jira / Zendesk
销售培训资料
内部SOP
```

当运营或客户成功遇到一个问题时，往往不知道应该去哪里查。

例如客户问：

> 为什么我的 iOS 安装数据在平台后台有，但是 MMP 没有？
> 
> 

这个问题可能需要同时查看：

```Plain Text
MMP归因规则
SKAN数据延迟机制
广告平台回传配置
客户App是否集成SDK
事件映射是否正确
归因窗口设置
客户是否开启ATT授权
历史类似工单
```

如果没有统一入口，人工排查会非常低效。



**问题二：投放问题复杂，依赖资深人员经验**

移动广告的异常问题往往不是单点原因，而是多个因素叠加。

例如 CPA 上涨，可能由以下原因造成：

```Plain Text
CPM上涨
CTR下降
CPC上涨
点击到安装CVR下降
安装到注册CVR下降
注册到付费CVR下降
素材疲劳
竞品加价
流量结构变化
归因延迟
MMP回传失败
App崩溃
支付异常
广告位质量下降
低质量流量涌入
```

新员工很难凭经验判断应该先查哪里。

资深运营通常会有一套隐性判断逻辑：

```Plain Text
先拆指标链路
再看异常发生时间
再看是否局部国家/系统/素材/广告位异常
再看是否平台侧和MMP同时异常
再判断是流量问题、素材问题、产品问题还是归因问题
```

但这些经验通常存在个人脑子里，没有被系统化沉淀。新人员工适应期较久，需要资深员工付出较大精力协助



**问题三：跨团队协作成本高，问题流转慢**

一个广告投放问题经常需要多个团队参与：

```Plain Text
客户提出问题
→ AM初步理解
→ 运营查看账户
→ 数据同学查BI
→ 技术支持查SDK或API
→ 产品同学看平台机制
→ 平台AM或媒体方确认外部原因
→ 再由AM回复客户
```

问题在多个角色之间流转，会带来明显的沟通损耗：

```Plain Text
客户描述不清
AM转述丢信息
运营重复要截图
技术需要补充日志
数据同学需要确认口径
最后回复客户时已经过去数小时甚至数天
```



**问题四：客户回复不标准，专业性不稳定**

同样一个问题，不同运营或 AM 给客户的解释可能不一致。

例如关于 MMP 和广告平台数据不一致，有人会说：

```Plain Text
“平台和MMP口径不一样，所以有差异。”
```

但这个回答过于笼统，客户很难接受。

更专业的回答应该包括：

```Plain Text
差异发生在哪个环节
可能原因有哪些
当前排查到哪一步
需要客户补充什么信息
预计什么时候给结论
后续如何避免
```

AI 可以帮助生成标准化、专业化、可追溯的回复。

### 可落地的 3 个子场景

#### 子场景一：投放效果异常诊断

用户问题示例：

```Plain Text
“昨天巴西安卓 Campaign 的 CPA 从 8 美元涨到 15 美元，帮我看下原因。”
```

AI 排查逻辑：

```Plain Text
1. 查看花费、曝光、点击、安装、注册、付费是否同时变化
2. 拆解 CPA = CPI / 安装到付费转化率
3. 拆解 CPI = CPC / 点击到安装转化率
4. 拆解 CPC = CPM / CTR
5. 判断是流量变贵、素材变差、落地页变差，还是后端转化变差
6. 检索历史类似案例
7. 输出建议
```

可能输出：

```Plain Text
初步判断 CPA 上涨主要来自 CTR 下降和素材疲劳。
过去 3 天 Top 3 素材曝光占比超过 70%，CTR 从 1.6% 降到 0.8%。
建议新增本地化素材，暂停 CTR 下降超过 40% 的素材，并观察 24 小时。
```

#### 子场景二：归因与数据不一致排障

用户问题示例：

```Plain Text
“客户说 MMP 看到 300 个安装，但平台后台只有 180 个，为什么？”
```

AI 排查逻辑：

```Plain Text
1. 检查双方统计时区是否一致
2. 检查归因窗口是否一致
3. 检查 MMP 与平台事件定义是否一致
4. 检查是否存在自然量、再归因、重复归因
5. 检查是否是 SAN 自归因口径差异
6. 检查 postback 是否延迟或失败
7. 检查 iOS SKAN 或隐私归因是否存在延迟
8. 检查数据是否按点击时间或安装时间统计
```

可能输出：

```Plain Text
当前差异可能来自统计口径不一致。
MMP报表按安装时间统计，平台后台按点击归因时间统计。
同时客户选择的是7天点击归因窗口，而平台侧显示为1天点击窗口。
建议先统一时区、时间口径和归因窗口后再对账。
```
