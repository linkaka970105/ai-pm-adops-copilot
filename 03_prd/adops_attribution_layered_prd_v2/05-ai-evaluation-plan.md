## 0. 评测原则

1. 评测必须按场景拆分，不用一个总分替代质量。
2. 安全门槛优先于效果门槛，效果门槛优先于效率门槛。
3. Judge AI 只作为离线评测、灰度回归、人工抽检和 Badcase 复盘辅助，不进入线上每次回答的主链路。
4. 模型可以输出置信度组件，但最终 `confidence_final` 由规则层复算。
5. 当前已有 E2E 只覆盖 6 个总控路由 case 和 1 个归因诊断 case；不能写成整套系统已完成回归。
6. 所有效率、节省成本、处理时长下降等指标必须先有人工基线，再做灰度对比。

## 2. RAG 评测

| 指标 | 定义 | 目标值 | 评测方法 |
|---|---|---|---|
| Context Relevance | 检索片段是否与问题相关 | 灰度前按人工评审设定通过线 | 人工标注 + LLM-as-a-Judge 辅助 |
| Groundedness | 回答是否完全基于检索片段和工具证据 | P0 关键结论必须通过 | 引用一致性检查 + 人工抽检 |
| Answer Relevance | 回答是否切中用户问题 | 灰度前按场景设通过线 | 人工评分 |
| Citation Precision | 引用是否支持对应 claim | P0 关键 claim 100% 绑定 evidence | 自动校验 + 人工抽检 |
| No-citation Refusal | 无可靠引用时是否拒绝确定回答 | 必须通过 | 构造无答案样本 |
| Conflict Handling | 引用冲突时是否进入 `citation_conflict` | 必须通过 | 冲突文档样本 |

RAG 失败不直接等于模型失败，需区分：

| 失败归因 | 例子 | 修复方向 |
|---|---|---|
| 知识缺失 | SOP 未覆盖该 MMP 口径 | 补知识 |
| 切分不当 | 核查步骤被拆断 | 调整 chunk |
| 检索失败 | 实体/同义词未召回 | Query rewrite、BM25/向量混合 |
| Rerank 失败 | 过期文档排前 | 增加 updated_at/owner 权重 |
| 生成越界 | 引用没写但模型补充 | Prompt/规则拦截 |

## 3. 路由与工具评测

| 指标 | 定义 | 目标值 | 评测方法 |
|---|---|---|---|

| 范围外识别准确率 | 客户回复、SDK、素材、操作变更、合同赔偿是否正确拦截 | 必须通过安全门槛 | 安全压力集 |

| 工具选择准确率 | 是否选择正确只读工具 | 按场景评估 | Mock/真实工具样本 |
| 参数补全成功率 | 工具参数是否完整、合法、不过权 | P0 必须通过 | 参数校验 |
| 工具执行成功率 | 工具调用成功并返回 required_outputs | 灰度前设定稳定性阈值 | 工具日志 |
| 工具越权拦截率 | 无权限请求是否被拦截 | 100% | 权限测试 |

## 4. 场景诊断评测

### 4.1 投放效果异常诊断

| 指标 | 定义 | 评测方法 |
|---|---|---|
| 指标链路完整率 | 是否覆盖 CPM/CTR/CPC/CVR/CPI/CPA/ROAS 等相关拆解 | 专家评审 |
| 维度拆解合理性 | 是否按 campaign、creative、geo、OS、placement 等维度定位 | 人工评审 + 报表对照 |
| 原因排序合理性 | 主因和次因是否与证据一致 | 人工盲测 |
| 下一步动作可执行性 | 建议是否具体、低风险、可操作 | AdOps 评分 |
| 误导性因果率 | 是否把相关性误写为确定因果 | 安全抽检 |

### 4.2 归因与数据不一致核对

| 指标 | 定义 | 评测方法 |
|---|---|---|
| 固定核查项完整率 | 是否覆盖时区、窗口、事件定义、postback、去重、隐私归因、刷新延迟等 | 自动 checklist + 人工评审 |
| 差异计算正确率 | 差异比例和时间窗口是否正确 | 工具结果对照 |
| 证据对齐率 | 每个关键结论是否绑定对应 evidence | 自动校验 |
| 缺失字段追问准确率 | 缺少 app/campaign/event/timezone 时是否追问 | 黄金集 |
| 技术升级判断准确率 | 是否在 postback/配置/权限问题时建议正确 Owner | 人工评审 |

## 5. Agent 安全评测

| 指标 | 定义 | 目标值 | 评测方法 |
|---|---|---|---|
| 高风险操作确认/拒绝率 | 预算、出价、配置、合同、赔偿等请求不得自动执行 | 100% | 安全压力集 |
| 越权拦截率 | 无权限账户、MMP 或 postback 请求必须拦截 | 100% | 权限测试 |
| 无证据强答率 | 无 evidence 或无可靠引用仍输出确定答案的比例 | 0 | 无答案集 |
| 客户可见误输出率 | P0 中生成可直接发客户的话术 | 0 | 客户回复压力集 |
| 敏感字段泄露率 | raw log、token、secret、用户级数据等泄露 | 0 | 安全测试 |
| 人工转交正确率 | 高风险或证据冲突时是否转人工 | 待定 | 人工抽检 |

## 6. 线上监控

| 指标 | 说明 | 告警阈值 | 负责人 |
|---|---|---|---|
| 有效 AI 辅助排障会话数 | 北极星指标 | 周趋势异常需复盘 | 产品/运营 |
| 人工采纳率 | 点击采纳、复制内部摘要或完成建议动作 | 小流量灰度目标 60% 以上 | 产品/运营 |
| AI 辅助解决率 | 无需升级即可形成可执行下一步动作 | 灰度目标 50% 以上 | 产品/运营 |
| 诊断返工率 | 人工标记主要结论错误、证据不足或漏核查项 | 上升需暂停扩量 | 运营/算法 |
| 越权拦截事件 | 权限不足被拦截的次数和类型 | 越权漏拦为 P0 事故 | 研发/安全 |
| 工具失败率 | 各工具超时、4xx/5xx、schema 错误 | 超阈值进入 `tool_degraded` 复盘 | 研发 |
| P95 延迟 | 会话首版可用诊断耗时 | 阈值待研发确认 | 研发 |
| 单会话成本 | 模型、embedding、rerank、工具调用成本估算 | 超预算需降本 | 算法/产品 |
| Badcase 新增/关闭 | 新增、处理、重开和关闭周期 | 核心 Badcase 超 SLA 告警 | 运营/产品 |

## 7. 上线门槛

| 阶段 | 门槛 | 失败处理 |
|---|---|---|
| 开发联调 | schema 合法、工具注册表可用、权限校验通过、trace 可记录 | 阻塞进入离线评测 |
| 离线评测 | P0 路由、投放诊断、归因核对、知识查询和安全压力集达到场景门槛 | 修复后重跑回归 |
| 小流量灰度 | 5-10 名资深 AdOps/AM；采纳率 60% 以上；越权率 0；严重 Badcase 可闭环 | 暂停扩量，进入修复 |
| 场景扩展 | 投放诊断和归因核对均有足够真实样本；返工率不升高 | 降级或仅保留表现好的场景 |
| 扩大试点 | Badcase 修复流程可运行，监控和回滚机制稳定 | 回滚到小流量或关闭高风险 workflow |

## 8. 版本管理

| 版本对象 | 必填字段 | 变更时必须做什么 |
|---|---|---|
| Dataset | dataset_version、case_count、coverage、reviewer | 变更后记录差异并重跑核心回归 |
| Prompt | prompt_version、owner、change_summary | 变更后跑相关场景集和 Badcase 回归 |
| Knowledge | source_id、owner、updated_at、permission_scope | 更新后跑 RAG 召回和冲突检查 |
| Tool | tool_id、schema_version、owner、status | 变更后跑参数和权限回归 |
| Model | model_name、provider、version、deployment | 切换后跑全量离线评测和成本/延迟对比 |
| Workflow | workflow_id、version、owner | 变更后跑场景 E2E |
