# 跨文档审查：AdOps Copilot 投放归因排障助手 v2

## 0. 审查结论

本版分层 PRD 已完成核心文档拆分：

- `00-prd-state.md`：状态、证据地图、已定决策、待确认问题。
- `01-decision.md`：业务目标、用户、范围、成功标准、非目标。
- `02-solution.md`：入口、实施计划、方案对比、灰度、风险和 ROI 口径。
- `03-implementation.md`：研发/测试可执行的功能、数据、接口、页面、验收和用例。
- `04-ai-capability-spec.md`：RAG、路由、工具、workflow、Prompt 契约、安全降级。
- `05-ai-evaluation-plan.md`：数据集、指标、场景评测、上线门槛和版本管理。
- `06-badcase-feedback-loop.md`：Trace、Badcase 漏斗、标注队列、回归集和运营节奏。

审查结果：可以作为 v2 草案进入人工评审。主要剩余风险不是文档结构，而是数据/API/权限/真实基线仍未确认。

| 旧版风险 | 本版处理 |

## 4. 幻觉与不准确风险审查

| 风险类型 | 当前处理 | 剩余风险 |
|---|---|---|
| 编造真实业务收益 | 所有效率和成本写成目标/公式/待验证 | 需要上线后用真实数据替换 |
| 编造已有 API | 工具注册表标为 P0 契约，待研发确认 | API 字段和权限仍可能调整 |
| 编造知识库覆盖 | 明确首批知识源待确认 | 知识质量需要人工治理 |
| 编造完整评测通过 | 已区分已有验证和待扩展验证 | 仍需补充投放诊断、知识查询、安全交付 E2E |
| 模型自报置信度当最终值 | 规定模型给 components，规则层给 final | 需工程实现保证 |
| 把相关性说成因果 | 文档要求“可能原因排序”和证据限制 | 需要评测集覆盖误导性因果 |
