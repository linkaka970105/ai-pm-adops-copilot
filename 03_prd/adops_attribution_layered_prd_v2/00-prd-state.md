| D-002 | 第一阶段 MVP 聚焦投放效果异常诊断、归因与数据不一致核对、知识查询和 Badcase 闭环 | 与 README 当前重点文档和旧版目标 PRD一致，范围更可控 | E-001, E-009 | 2026-07-01 |
| D-003 | SDK/API 深排、素材审核、客户可直接发送回复、预算/出价/配置变更暂不纳入本版 P0 | 这些场景风险、数据和流程差异较大，容易让 MVP 失焦 | E-009, E-010, E-013 | 2026-07-01 |
| D-004 | AI 输出必须证据化，业务结论必须绑定 Evidence Object；无可靠证据时不输出确定性结论 | 产品场景涉及客户和投放决策，幻觉成本高 | E-005, E-015, E-016 | 2026-07-01 |
| D-005 | 工具调用只读优先，第一阶段不自动改预算、出价、账户配置或对外发送客户回复 | 高风险动作需要人工确认，MVP 先验证诊断价值 | E-010, E-014 | 2026-07-01 |
| D-006 | Query rewrite 归属于总控/路由层，产出检索计划和工具计划，不直接生成业务结论 | 避免检索改写和诊断生成职责混乱 | E-014, E-017 | 2026-07-01 |
| D-007 | Judge AI 不作为线上每次回答的强制拦截器，主要用于离线评测、灰度回归、人工抽检和 Badcase 复盘 | 降低线上延迟和误拦截风险，同时保留质量闭环 | E-017 | 2026-07-01 |

## 5. 术语表

| 术语 | 定义 | 来源 |
|---|---|---|
| AdOps | 广告运营，负责投放配置、异常排查、效果跟进和跨团队协调 | E-003, E-004 |
| MMP | Mobile Measurement Partner，移动归因和营销测量平台，如 AppsFlyer、Adjust 等 | E-005, E-011 |
| 归因差异 | 广告平台、MMP、客户 BI 对同一转化事件统计口径、时间、窗口或数据链路不一致导致的差异 | E-005, E-009 |
| Postback | MMP 或客户系统向广告平台回传转化事件的机制，本版仅查询聚合摘要，不处理 raw URL 或用户级日志 | E-009, E-014 |
| RAG | 检索增强生成，用于从 SOP、文档、历史工单和口径材料中取证后生成回答 | E-008 |
| Evidence Object | 统一证据对象，承载事实、来源、可见性、Owner、检索时间和可信度 | E-015 |
| Delivery State | 前端交付状态，用于表达回答可用、部分证据、工具降级、无引用、权限不足、范围外或系统错误 | E-016 |
| Badcase | AI 回答、路由、工具调用、证据引用或交互流程中的失败样本，包括显性错误和高成本/低体验的隐性失败 | E-008, E-017 |
| 黄金数据集 | 用于离线评测和回归验证的标准样本集，覆盖高频、边界、权限、工具失败和多轮追问场景 | E-017 |
